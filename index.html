Version control

Version control

is a way to:

  • Back up changing files
  • Store and access an annotated history
  • Manage merging of changes between different change sets

Version control

Categorization 1

  1. Centralized: Subversion (svn)
  2. Distributed: Git (git)

Version control

Categorization 2

  1. Local: Git
  2. Remote: GitHub
